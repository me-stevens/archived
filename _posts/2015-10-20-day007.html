Today I learnt

In today's Zagaku, Christoph tried to explain us how to implement the Open-Closed Principle or OCP, where you write your code in a way that your classes are open to extension but close to modification. This means that you don't have to modify the code you have already written if you want to add new features (close to modification), you just add them, which means, you write new code (open to extension).

It reminded me of the blog post Alternatives to boolean parameters by Dariusz Pasciak, which is not exactly about the OCP in particular, but the solution it proposes to his problem follows this principle. I found the example to be so good when I first read it because you always hear about all these concepts, but never see them implemented with a clear example.

After the Zagaku I paired with him and finally solved the roman numerals kata. It's such a simple kata, and yet I had it all screwed up because I can't seem to be able to think in a simple way. Anyway, I was so happy that I gave him a huge hug. I really have to learn to think the right way or I will kill myself.
