Today I learnt

Christoph talked about learning a new language. He asked us which language were we learning at the moment if any, and what were we doing to learn it. Katas, koans, writing code, etc. started to pop out.

Christoph recommended that we should also keep an eye on the community, to learn how they do things. For example, what do people do to start a project in Java? They probably use Maven to generate the folder structure. What do people use to manage Ruby versions and packages? Probably rbenv, and so on.

He also talked about language versions and dependencies. I learnt that you can use two different unit test libraries and that doesn't mean you are bloating your codebase, because tests are not deployed.

The Zagaku finished and I went to learn about the Optional class in Java 8. It's something you would use to avoid null pointer exceptions when you try to, for example, call a method on an object that hasn't been created yet, or access an element of an array that doesn't exist.

The optional class has a set of methods that allow you to write Java in a more compact way, for example, you don't have to fill your codebase with null checks, because the Optional class takes care of that at compile time, with methods like ifPresent().

Challenges

I have to measure the time a story takes me, to learn how to estimate time better. The problem with time estimations is that there are things popping up in this office, interruptions, activities, and what not. And those are totally out of my control. It's not even as estimating a background contribution in your measurement, because backgrounds are more or less well known and can be modeled, and they are usually more or less "constant", i.e. they don't have peaks or surprises. But this office... THIS OFFICE IS SO ALIVE. It would be like trying to model chaos. After all, people still can't predict the weather accurately a month in advance.

Nice things
