Today was a weird Friday. For the first time I didn't have to finish any of my stories, and I had the whole Friday to focus on my waza.

There's something I've been trying to do since before starting the apprenticeship. I want to learn to use public APIs like those of GitHub, Twitter or Youtube, to fetch and send data. As usual, I never have the time to sit and work on that. So I tried to do that today.

I worked with the GitHub API. I want to build a webapp where users can log-in to GitHub, and fetch data. Then, they would do stuff with the data. Then the stuff they did would be persisted in a GitHub gist.

This requires several things. The log-in I left for later, because I have to learn about OAuth, but learning this will take me forever.

I also want to authenticate in the client side, but the GitHub API only lets you do it server side. I guess I could deploy to Heroku and use PHP or whatever, as Lea Verou does in her webapp dabblet, but I wanted something simple that I can use in GitHub pages. I suspect I will have to use PHP or similar and deploy to a proper server.

The second part is to fetch the data and get what I want from the JSON object sent by GitHub. I managed to make a spike with HTML and vanilla JavaScript that is working, but the code makes your eyes bleed and there are no tests. Also, I'm hard-coding my user-name, lol. L'horreur.

The third part is to send data to GitHub and save it in a gist. For this I guess I would need to have the log-in part finished first.

Finally, I tried to add tests, for which I used Jasmine. But I had no idea how to test an HTTP connection like that. Then I saw Rabea learning about Sinatra for her stories, and she was testing GET and POST requests to localhost, which is similar to what I want to do. However, I don't know if I want to use Sinatra for this particular thing or if it would even work.

I've found that Jasmine comes with an AJAX library to mock AJAX responses called Jasmine-AJAX, so I may try that.

What I know for sure is that I don't want to use a trillion frameworks. Dabblet works with only a bunch of vanilla JavaScript files. Part of the reason why I'm doing this is to understand what's going on, as opposed to having a lot of magic doing things for me or different technologies fighting each other.

So I idn't do much today... even though I have a working... "something". Working as in, it connects to GitHub, fetches user and repos data, and spits them in the html. Nothing, basically. A day might not be enough for me to finish this... I foresee the need of at least a month!
