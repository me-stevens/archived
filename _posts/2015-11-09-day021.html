Today I learnt

In today's Zagaku, Felipe explained the difference between Stacks and Heaps. Stacks are like memory frames that are created and destroyed on the fly, every time a program makes a function call. The variables are local to the frame and don't change their values while living in the stack. Stacks are nice because they allocate a very small piece of memory, which is liberated fast. A stack overflow is caused when you overpass the maximum number of stacks that the system can create, which is around 65000 in Java.

Heaps are bigger, they are usually of the order of Gigabytes. Since their use of space is not efficient, they are slower, because the computer has to play Tetris with the size of the elements in the heap, and the available space gets very fragmented. Also, you have to do all the memory management manually. In C, that would be done with calls to malloc() and calloc(), for example, which then you will free with free(), as opposed to a stack, where the system takes care of the garbage collection for you.

Sara mentioned our last week's issue, when we asked for the difference between a stack overflow and an infinite while loop. The infinite while loop stays in its frame forever, and that doesn't cause the system to explode. However, it's very rare to reach the so many stacks needed for that. But it's a good practice to use the while loop instead of a recursive method!
