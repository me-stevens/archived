Today I learnt

It was Felipe's Zagaku time, and he used yesterday's exercise (the one where I screwed up) to teach us what a good pull request (PR) looks like. In short:

  • Create a new branch and commit your changes to it.
  • Write concise and clear commit messages.
  • Check that all the tests are passing before you push.
  • It's ok to make a PR of something that is not complete yet, just explain it in you message in a clear way.
  • It's ok to rebase some commits before making the PR, if they are too granular, so that the reviewer of the PR is not faced with 30 commits all of a sudden.
  • Once the PR is placed, in GitHub (and I think also in BitBucket and GitLab), you can tag other users if you want them to review the PR.
  • When agreeing with a PR, it is considered negative to just say "It's ok". Because in written communication you lack things like tone, body language and so on, it sounds neutral, even if it is positive. So you have to... add an emoji! I kind of like this one but I fear of me abusing it.
  • LGTM = "Looks good to me", is something you can also use as a comment when you agree with the commits.
  • If the discussion on a PR is more than 5 comments long, move it somewhere else, for example discuss it in person.

He asked us to practice during the following days making PR's and commenting them.
