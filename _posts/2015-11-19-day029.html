I honestly learnt a lot, and refreshed concepts I already knew.

This packed-like wisdom shown in a direct way is a format that really works for me.

This is what I took from the first video:

  • Name your variables properly, so that they communicate their intent and don't misinform.
  • The cost of software is its maintenance.
  • Use nouns to name classes and variables, but don't use too general names like "info" that mean nothing.
  • Use predicates to name booleans, like isSet or hasWinner. Same for functions that return a boolean.
  • Use the verb get to name accessors.
  • Properties, like those in C#, are methods pretending to be variables, so use nouns (or predicates if they are boolean).
  • Enums represent state, so use and adjective.
  • Regarding scope: for variables, the shorter the scope, the shorter the name (for example, the iteration variable in a for loop can be just one letter long). Longer scope variables need a more descriptive name so that you don't have to go 20 lines up to see what it is.
  • The opposite is true for functions and classes, the bigger the scope the more places you will be using them, so use short names like open() in the File class.

And from the second video:

  • A function should do one thing, well and only. Or as he said, first, should be small, second, should be smaller than that. Four or five lines is ok, ten is too long.
  • If you have a long condition in an if statement, it's a good idea to extract it into a method.
  • "A long method is where all the classes were to hide". If you see one, it can probably be extracted into a class. The local variables used everywhere would be fields that we'll pass in the constructor. You can even move some of those fields to local variables inside certain methods.
  • The bedroom metaphor: a 12 year old would have everything scattered. Keep your room clean! Put your code in nicely named small places so that other people can find them.

It really comes down to "use common sense", but its always cool to have these tips present in the back of your head while you are coding.
